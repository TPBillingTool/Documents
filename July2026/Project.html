Evolutionary Delivery: Building on Rock

To mitigate the risks of global "big bang" deployments, we use an Evolutionary Delivery model.

  • Phase 1: The Host Shell acts as our foundational rock, managing authentication and global routing.
  • Phase 2: Independent country logic (MFEs for Colombia and Peru) is dynamically plugged in, achieving zero global downtime.

Enterprise Monorepo Ecosystem

Nx Orchestration + Native Federation: Massive scalability with strict dependency isolation.

The Shell orchestrator dynamically loads remotes without tight coupling. Shared libraries (shared-domain, shared-infra-ui) guarantee that core models never depend on country-specific code, mitigating cascade failures.

The Technology Engine

An enterprise-grade stack perfectly suited for robust financial math.

  • Frontend (Angular): Ensures strict typing and Native Federation for dynamic module loading.
  • Backend (Django): Built-in protection against OWASP Top 10 vulnerabilities (SQLi, XSS, CSRF) and robust ACID-compliant transaction management.

Hybrid Hexagonal Architecture

Driven by Domain-Driven Design (DDD), external systems (ERPs, IdPs) can be swapped without ever breaking the core financial math.

  • Pure Business Logic: Provision calculations and invoice transitions.
  • Application Ports: Orchestration of use cases.
  • Infrastructure Adapters: Django Persistence, D365 ERP adapter, OAuth2.

Contextual Workflows: Local Complexity

Both paths utilize the exact same backend core, enforcing local compliance effortlessly.

  • Path A: Colombia. Manual flexibility. User inputs descriptions and executes manual calculations.
  • Path B: Peru. Highly restricted. Pre-loaded SalesAgreements lock descriptions and fixed rates; operations only edit quantities.

The Impenetrable Vault (Zero-Trust)

For a financial platform of this caliber, security is paramount. Implemented via a 5-Layer vault:

  • Layer 1: Forced TLS 1.3 & HSTS in transport.
  • Layer 2/3: Stateless JWT RBAC validation (OAuth 2.0 + PKCE) & Application Double-Check.
  • Layer 4: Django Security Middleware (CSRF) & Angular DomSanitizer (XSS).
  • Layer 5: AES-256 encrypted configuration data persistence.

Absolute Data Sovereignty

To guarantee compliance with data laws across 130+ nations, we use a strict DB-per-Tenant physical isolation model.

The Tenant Resolver extracts the country claim from the JWT, passing it to the Dynamic DataSource Router which mathematically prevents cross-tenant query leaks.
